Flask WebAPI 学習用アプリ 利用手順書
1️⃣ 本書の目的
この手順書は、Flask WebAPI 学習用アプリを利用して、ブラウザ上から簡単にAPIの動作確認を行うための手順を説明します。

2️⃣ 前提条件
アプリが正常に起動できる状態であること（詳細は README.md を参照）

ブラウザ（Google Chrome または Edge）を使用可能であること

3️⃣ 画面構成
アプリを起動後、以下のページが表示されます。
http://localhost:5000/
画面には3つのAPIテスト機能が用意されています。

項目	説明
1. アクセステストAPI	サーバーが動作しているか確認できます。
2. エンドポイント一覧API	提供されているAPIのURL一覧を取得します。
3. JSONエコーAPI	入力したJSONデータをPOSTし、同じ内容を返すことを確認できます。

4️⃣ 操作手順
4-1. アプリの起動
コマンドプロンプトまたはターミナルを開く

アプリのフォルダへ移動

以下を実行
python flask_test_app_250801.py
「Running on http://127.0.0.1:5000/」と表示されたら起動成功

4-2. ブラウザでアクセス
ブラウザのアドレスバーに以下を入力
http://localhost:5000/
Webページが表示される

4-3. 各機能の利用方法
(1) アクセステストAPI
「実行」ボタンを押す

結果欄に以下のようなメッセージが表示される
{
  "message": "API接続成功"
}

(2) エンドポイント一覧API
「実行」ボタンを押す

結果欄に以下のように表示される
{
  "endpoints": [
    "/test",
    "/endpoints",
    "/echo"
  ]
}

(3) JSONエコーAPI
テキストエリアに以下のようなJSONデータを入力
{
  "name": "Taro",
  "age": 30
}
「実行」ボタンを押す

結果欄に同じデータが返される
{
  "received_data": {
    "name": "Taro",
    "age": 30
  }
}

5️⃣ トラブルシュート
問題	対処方法
ページが開けない	サーバーが起動しているか確認（Ctrl+Cで停止後、再度起動）
ボタンを押すと「Failed to fetch」と出る	Flaskにflask-corsを導入してCORSを許可。または同一ポートから提供
JSONエコーでエラーが出る	入力が正しいJSON形式か確認。キーと値のダブルクォート必須

6️⃣ 注意事項
本アプリは学習目的専用です。本番環境での使用は推奨されません。

APIエンドポイントやHTMLはローカル環境のみで利用してください。
